- byte -- 1 bit；short -- 2 bit；int -- 4 bit；long -- 8 bit

- boolean -- 无；char -- 2 bit

- float -- 4 bit；double -- 8 bit


### 2.面向对象的特征？

面向对象三大特性：继承、封装、多态。

继承：是指子类继承父类的功能和方法，复用父类的代码。

封装：将一个对象的属性私有化，对外界提供访问属性访问方法。

多态：在程序中声明变量，变量指向其类型的子类或者实现类，在编译器不知道具体实现对象，只有在运行期知道具体实现类。

### 3.面向对象和面向过程区别？

面向过程关注的是"具体怎么做"，而面向对象关注是“让谁去做“，而这个谁就是指对象，不具体关注被调用者怎么去实现，许多对象共同完成一个功能点。

### 4.final, finally, finalize 区别？

final：用来修饰类、常量、方法，修饰类则不能被继承、修饰常量则初次赋值后则不能改变、修饰方法则方法不能被子类继承重写。

finally：用在catch代码块后面，无论程序是否抛出异常都会执行该代码块。

finalize：是Object类中的一个方法名，Java允许程序中调用此方法进行该对象的回收，但是不代表调用就会立即执行，只有当垃圾收集器空闲时且确定该对象可被回收时才会回收。

### 5.Error, Exception, RuntimeException 区别？

他们都继承自Throwable类，Throwable子类分为两大类，一类是Error代表JVM无法处理的错误，如OutOfMemoryError内存溢出和StackOverFlowError栈溢出，另一类是Exception异常；而Exception又分为两类，一类是可捕获异常，例如IOException，另一类是RuntimeException运行时异常，也就是不可捕获异常，例如空指针异常和数组下标越界异常。

### 6.5种常见的RuntimeExcetpion?

- 空指针异常：NullPointerException
- 字符串索引越界异常：StringIndexOutOfBoundsException
- 数组索引越界异常：ArrayIndexOutOfBoundsException
- 强制类型转换异常：ClassCastException
- 找不到类异常：ClassNotFoundException

### 7.int 和 Integer 区别？ Integer 缓存范围？

int是基本数据类型，Integer是int的包装类类型。Integer封装了许多操作int的方法。Integer在使用时需要去创建对象才可以使用。

Integer缓存范围为-128~127。

### 8.包装类, 装箱, 拆箱？

8种基本数据类型都有对应的包装类类型，除int的包装类为Integer和char的包装类为Character外，其他6种基本数据类型的包装类为将其首字母大写。

自动装箱：是在JDK5之后，程序中无需手动调用方法实现类型转换，而是自动转换，通过反编译工具查看源码底层设计上是调用封装类的valueOf方法。

自动拆箱：底层实际上是调用封装类的xxxValue方法。

### 9.String, StringBuilder, StringBuffer 异同？

相同点：它们都是操作字符串的类。

不同点：

可变性，在JDK8中，String底层使用final修饰char数组，所以String是不可变的字符串，一旦初始化，就不能被改变，而StringBuilder和StringBuffer没有使用final修饰，所以是可变的。

线程安全性，由于String是不可变的，所以它是线程安全的；StringBuffer底层方法使用synchronized关键字枷锁，而StringBuilder没有，所以StringBuffer是线程安全的，而StringBuilder是线程不安全的。

性能方面，String适合操作少量数据，StringBuilder用在大量数据且单线程条件下性能比较高，而多线程只能使用StringBuffer。

### 10.String 不可变的好处？

在多线程环境下，保证了的安全性；

String普遍用做参数传递，他不可变的特性，保证了参数的安全性。

String不可变，所以一个字符串的散列值，只需要一次计算。

new的时候在堆中创建String对象。

### 12.重载和重写区别？

重载：发生在同一个类中的，重载指的是方法名相同而参数类型、参数列表和返回值不同的同名方法。

重写：发生在继承体系的子类中，在子类中重写父类方法。

### 13.抽象类和接口区别？

在组成成员上，抽象类成员变量可以是常量也可以是变量，而接口只能是常量；抽象类有构造方法，而接口无构造方法；抽象类的方法可以是抽象的也可以是具体的，而接口只能是抽象的，在JDK8时default关键字修饰的方法可以有方法体。

在关系上，类与类只能单继承，类与接口可以多实现，接口与接口可以多继承。

在设计理念上，抽象类表示共有的属性和功能，而接口表示的是扩展性功能。

### 14.反射的用途及实现？

反射的主要用途在框架中。

通过Class类的forName方法，获取类的Class对象，然后通过Class对象便可以获取到类的构造器，进行初始化对象。

### 15.equals 和 == 区别？

==：比较的是基本数据类型值是否相等，比较的引用类型的地址值时候相等。

equals：在Object类中默认是使用==进行比较，重写之后，比较两个对象时候相等。

### 16.hashCode 和 equals 区别和联系？

在底层涉及到散列的类中，如HashMap在比较两个元素时候相等时，先比较两个元素的散列值，也就是Hash值是否相等，然后在调用equals比较元素的内容时候相等。

### 17.Java 的序列化和反序列化？

类要想序列化存储到本地硬盘，需要实现Serializable接口，这个接口是标记接口，其中没有任何方法，只是标记的作用，之后再使用ObjectOutputStream类将对象序列化到本地。如果想要读取序列化的类，使用ObjectInputStream类将其读取到内存。

### 18.为什么 wait() 和 notify() 属于 Object类？

synchronized中锁为任意对象，所以任意对象都要有wait和notify方法，而Object是所有类的父类，所以wait和notify属于Object类。

### 19.JDK, JRE, JVM 区别与联系？

JDK java开发套件；JRE java运行环境，其中包含基本的类库和工具；JVM java虚拟机，用来执行class文件。

JDK包含JRE，JRE包含JVM。

JDK除了包含JRE之外，它还包含许多开发工具，如编译器、调试器等工具，而JRE只是运行环境，不包含开发工具。

JRE除了包含JVM之外，它还包含Java代码运行时基本的类库。

### 20.浅拷贝和深拷贝区别？

浅拷贝：是只拷贝类变量的地址。

深拷贝：拷贝类变量对象。

### 21.静态变量, 静态代码块, 实例变量, 普通代码块, 构造方法初始化顺序和继承情况下顺序？

静态变量和静态代码块顺序取决于代码中先后顺序。

静态变量和代码块属于类，变量和代码块属于对象。

在继承情况下的执行顺序：

1）父类（静态变量，静态代码块）

2）子类（静态变量，静态代码块）

3）父类（实例变量，普通代码块）

4）父类（构造器）

5）子类（实例变量，普通代码块）

6）子类（构造器）

### 22.无参构造函数作用？

### 23.成员变量和局部变量区别？

从代码上看，成员变量属于类，局部变量属于方法。

从内存上看，如果成员变量用static修饰，则成员变量属于类，没有被static修饰则属于示例对象；局部变量属于Java栈。

成员变量有默认初始化值，而局部变量没有，使用前需要被初始化。

### 24.构造方法特性？

没有返回值，方法名和类名相同，无返回值，不写构造方法会有默认无参构造。
